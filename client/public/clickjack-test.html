Clickjacking Prevention Demonstration

This page demonstrates clickjacking prevention.

The iframe below attempts to embed the CS418 application login page. Because the server sends X-Frame-Options: DENY and Content-Security-Policy: frame-ancestors 'none', the browser refuses to render the page inside the frame.

Note: In production, the app is served by Express which sets these headers. In local development (React dev server on port 3000), headers are set by Express on port 5000 when serving the built app.

How clickjacking prevention works:

  • X-Frame-Options: DENY — Legacy header that instructs browsers to never display this page inside any <frame>, <iframe>, <embed>, or <object> element.
  • Content-Security-Policy: frame-ancestors 'none' — Modern CSP directive that supersedes X-Frame-Options. Prevents any origin (including same-origin) from embedding this page in a frame.

Without these headers, an attacker could:

  • Embed the login page invisibly over a malicious site
  • Trick users into clicking buttons they cannot see (UI redressing)
  • Steal credentials or perform unintended actions on behalf of users
